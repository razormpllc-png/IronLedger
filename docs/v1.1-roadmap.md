# Iron Ledger v1.1 — Roadmap

Tracked bugs, UX fixes, and feature requests for the next release.

---

## Bug Fixes

### Keyboard blocks bottom input fields
- **Screens affected:** Any form with notes/conditions fields near the bottom (DOPE card conditions, range session notes, expense notes, firearm notes)
- **Issue:** When the keyboard opens for a multiline TextInput near the bottom of a ScrollView, the field scrolls behind the keyboard and the user can't see what they're typing
- **Fix:** Replaced KeyboardAvoidingView with `FormScrollView` component using `automaticallyAdjustKeyboardInsets` (native RN 0.73+ prop). Applied across all 17 form screens.
- **Priority:** High — ✅ Completed April 22, 2026

### Auto-save on navigate away / screen lock
- **Screens affected:** All add/edit forms (firearm, accessory, suppressor, ammo, DOPE card, range session, expense)
- **Issue:** If the user partially fills a form and then navigates away (swipe back, tap a notification, phone locks), all entered data is lost
- **Fix:** Implement auto-draft persistence. Options:
  1. **AppState listener** — save form state to AsyncStorage on `inactive`/`background` events, restore on re-mount
  2. **useEffect cleanup** — persist draft on unmount (less reliable on iOS due to suspension)
  3. **Debounced auto-save** — write to AsyncStorage on every field change with a 500ms debounce, clear on successful save or explicit cancel
- **Recommended approach:** Option 3 (debounced auto-save) is the most bulletproof. Key the draft by screen name + editing ID so multiple drafts don't collide. Show a subtle "Draft saved" indicator. On re-open, detect existing draft and prompt "Resume where you left off?"
- **Priority:** High — ✅ Completed April 22, 2026. Implemented `useAutoSave` hook with debounced AsyncStorage + AppState background flush across all add screens.

## UX Improvements

### Tap-to-expand photos
- **Screens affected:** Firearm detail, accessory detail, suppressor detail — anywhere additional photos are displayed
- **Issue:** Additional photos are shown as small thumbnails but there's no way to tap and view them full-screen
- **Fix:** Add a full-screen image viewer modal. Options:
  1. `react-native-image-viewing` — lightweight, supports pinch-to-zoom and swipe-to-dismiss
  2. Custom modal with `Animated` + `PanResponder` for swipe-to-dismiss
- **Recommended:** Use `react-native-image-viewing` — well-maintained, handles gestures and status bar correctly out of the box. Wrap the thumbnail in a `TouchableOpacity` that opens the viewer with the tapped image as the initial index.
- **Priority:** Medium — ✅ Completed. Custom full-screen modal viewer with navigation arrows on firearm detail screen.

## New Features

### Export / Print DOPE cards
- **Description:** Allow users to export DOPE card data for offline reference (printed range card) or backup
- **Export formats:**
  1. **Single DOPE card** — export one card as a formatted PDF (range-card style layout with distance rows, elevation, wind holds, card metadata) or as a CSV with columns: Distance, Elevation, Windage, Notes
  2. **All DOPE cards (CSV)** — bulk export every card across all firearms into a single CSV file. Columns: Firearm, Card Name, Ammo, Zero, Units, Distance, Elevation, Windage, Notes
- **Implementation approach:**
  - CSV: Build string in JS, write via `expo-file-system`, share via `expo-sharing`
  - PDF: Use `expo-print` to render an HTML template to PDF, then share. The HTML template should look like a traditional DOPE card (dark header, grid rows, metadata block)
- **UI:** Add a share/export icon button in the DOPE card detail screen header. Tapping it shows an action sheet: "Export as PDF", "Export as CSV", "Cancel"
- **Bulk export:** Add "Export All DOPE Cards" option in a settings or data management screen
- **Priority:** Medium — ✅ Completed April 22, 2026. Single-card PDF + CSV export via share sheet (⇪ button in header).

### Armory screen — color-code firearm types
- **Screen:** Armory tab (firearms list)
- **Issue:** All firearms look the same at a glance. The "condition" field currently stands out in a different color but doesn't need to — it's not useful as a visual differentiator
- **Fix:**
  1. Remove the distinct color on the condition badge/text — let it blend with the rest of the card
  2. Add a subtle color accent per firearm type so you can tell Pistol / Rifle / Shotgun / Other apart instantly. Apply the color to the type badge, a left-border stripe, or an icon tint — not the entire card
  3. **Do NOT use green or red** (those read as "good/bad" and conflict with condition semantics)
  4. **Do NOT change suppressor card colors** — keep suppressors as-is
- **Suggested palette (adjust during implementation):**
  - Pistol → Blue (#4A90D9)
  - Rifle → Orange/Amber (#D4912A)
  - Shotgun → Purple (#8E6FBF)
  - Other → default muted gray
- **Priority:** Medium — ✅ Completed. TYPE_COLORS map with Handgun (blue), Rifle (orange), Shotgun (purple) applied to type badges on armory cards.

### Transfer documents — Bill of Sale / Receipt
- **Screens affected:** Dispose (transfer out), transfer-in flow
- **Description:** When transferring a firearm out, offer an option to generate a **Bill of Sale** document. When transferring in, offer an option to generate / attach a **Receipt**. Both should be shareable as PDF via the system share sheet.
- **Bill of Sale template:** Seller name, buyer name, date, firearm details (make/model/serial/caliber), price, signatures block, state disclosure text
- **Receipt template:** Seller/source, date, firearm details, amount paid, transaction reference
- **Implementation:** Use `expo-print` to render HTML → PDF, then `expo-sharing`. Pre-fill from firearm record + user profile. Editable fields for counterparty info and price.
- **Priority:** Medium — ✅ Completed April 22, 2026. Bill of Sale PDF with shared lib/billOfSale.ts, available from dispose screen + firearm detail disposition card.

### Round count per firearm
- **Screens affected:** Firearm detail screen, Armory list cards
- **Description:** Display a cumulative **round count** on each firearm showing total rounds fired through that platform. Derived from range session logs (rounds fired per session linked to that firearm). Show on the firearm detail screen as a prominent stat and optionally as a subtle line on the Armory card.
- **Data source:** Sum of `rounds_fired` from range sessions where `firearm_id` matches. If manual adjustments are needed (e.g. rounds fired before the app), add an optional `initial_round_count` field on the firearm record.
- **UI:** Firearm detail → stat row showing "Rounds: 1,247" alongside other key stats. Armory card → small muted text like "1.2k rounds" under caliber.
- **Priority:** Medium — ✅ Completed. round_count column + getRoundCountForFirearm() function + display on armory cards and firearm detail.

### Personal vs Business firearm designation (FFL module)
- **Screens affected:** Add firearm, edit firearm, estate planning, inventory, insurance output, all list/filter screens
- **Description:** When the FFL module is installed, prompt the user during firearm entry: **"Is this a Personal or Business firearm?"** Store as a `ownership_type` field (`personal` | `business`). This designation should propagate to all areas that output or filter inventory — estate planning should only include personal firearms by default, insurance reports should allow filtering by ownership type, and the Armory should support a filter/tab toggle.
- **Implementation:** Add `ownership_type TEXT DEFAULT 'personal'` column to firearms table. Add toggle/chip on add-firearm and edit-firearm screens (only visible when FFL module is active). Add filter support to all query functions. Estate planning defaults to personal-only with option to include business. Inventory/insurance screens get a filter chip row.
- **Priority:** Medium — ✅ Completed April 22, 2026. ownership_type column + Personal/Business toggle on add/edit firearm + display on detail screen.

### Bulk import — CSV / spreadsheet import
- **Screen:** New `/import` screen, accessible from Armory + button → "Import from Spreadsheet"
- **Description:** Three-step flow: pick file → map columns → preview & confirm. Supports CSV and TSV files. Smart auto-mapping guesses which columns are make, model, caliber, serial, etc. User can override any mapping before importing. Shows preview cards for all rows before inserting. Handles quoted fields, dollar signs in prices, commas in numbers.
- **Formats:** CSV, TSV, TXT (tab-delimited). Excel (.xlsx) prompts user to export as CSV first (avoids heavy SheetJS dependency for v1).
- **Files:** `lib/importParser.ts` (parser + field mapping), `app/import.tsx` (UI)
- **Priority:** Medium — completed April 21, 2026

### Competition Module (future expansion)
- **Description:** Full competition tracking for USPSA, IDPA, Steel Challenge, and outlaw matches. Dedicated module for competitive shooters.
- **Core features:**
  - Match type toggle: **USPSA / IDPA / Steel Challenge / Outlaw** — adapts form fields per discipline
  - Quick PractiScore URL paste with auto-extract of match name
  - Per-stage scores, hit factors, and stage breakdowns
  - Number of stages per match
  - Placement tracking with division/class context
  - Gun used + ammo used per match (linked to Armory + Supply)
  - Squad notes
  - Range location (linked to existing location data)
  - Video upload or screenshot of scoresheets
  - Match history / stats dashboard showing trends over time
- **USPSA-specific fields:**
  - Hit factor per stage, points, A/C/D/M counts, time, penalties
  - Classifier percentage, division (Production, CO, Open, Limited, SS, Revolver, PCC)
  - Class (GM / M / A / B / C / D)
- **IDPA-specific fields:**
  - Points down, time + penalties, stage score
  - Division (CCP, SSP, ESP, REV, CO, PCC)
  - Classification (DM / MA / EX / SS / MM / NV)
- **Results import — 3 approaches:**
  1. **PractiScore CSV import** — match directors can export results as CSV. Reuse the column-mapping import flow to pull stage scores, HF, penalties, time, placement.
  2. **URL paste + parse** — paste a PractiScore match URL, fetch the page, find the shooter's row, extract stage-by-stage data automatically. Fragile but great UX when it works.
  3. **Screenshot capture** — store a screenshot of the PractiScore app or printed scoresheet as a photo on the match record for visual reference.
- **Implementation:** New `competition_matches` and `competition_stages` tables. Separate tab or sub-section within Range. Can share the range session record or exist independently for matches not shot at a "practice" range trip.
- **Priority:** Medium — ✅ Completed April 22, 2026. Full competition module with `competition_matches` + `competition_stages` tables, add/edit match screen with type-specific fields, match detail screen with per-stage score entry (A/C/D/M hits, HF, time, penalties), and "Matches" segment on Range tab.

### Ammo — reloading load data
- **Description:** Load data tracking for handloaders, especially rifle and competition shooters. Fields:
  - Powder: brand, type, charge weight (grains)
  - Bullet: brand, weight, type (e.g. "Hornady 77gr BTHP")
  - Brass: brand, times fired
  - Primer: brand/type
  - COAL (Cartridge Overall Length)
  - CBTO (Cartridge Base to Ogive)
  - Velocity (fps), SD, ES from chronograph
  - Group size (MOA or inches at distance)
  - Load notes / development notes
- **Ties to:** DOPE cards (a load → a DOPE card), range sessions (which load was used), ammo inventory (track handloaded lots vs factory)
- **Implementation:** Either extend the `ammo` table with nullable reload columns, or create a `load_recipes` table that links to ammo lots. Add an "is_handload" flag on ammo lots that reveals the load data fields.
- **UI:** Toggle on add-ammo screen: "Factory" vs "Handload". Handload reveals the load data card. Load recipes could be saved and reused across batches.
- **Priority:** Medium — ✅ Completed April 22, 2026. Factory/Handload toggle with 7 LOAD DATA sections (18 fields) on add/edit ammo screens + DB schema.

- **Priority:** Medium — ✅ Completed April 22, 2026. Type chips (Flat, Angled, Flash Hider, Sacrificial, None) + notes on add/edit/detail screens.
